# Security Overview — Copilot Studio Agents for Marketing Teams

How we keep data safe when deploying AI agents in a Microsoft enterprise environment.

---

## Authentication

| Component | Authentication Method | Notes |
|-----------|---------------------|-------|
| Copilot Studio agents in Teams | **Microsoft Entra ID (automatic)** | Users authenticate via their M365 credentials. No separate login. |
| Power Automate flows | **Entra ID + connection credentials** | Each connector authenticates independently. |
| MCP servers (Work IQ) | **Managed by M365 admin** | Enterprise-grade, tenant-scoped permissions. |
| MCP servers (GitHub) | **Personal Access Token** | Scoped to specific repos. Stored as a secret. |
| MCP servers (Dataverse) | **Entra ID** | Uses existing Dynamics 365 permissions. |

## Data Classification

### What the agents CAN access (by design)

| Data Type | Source | Classification | Notes |
|-----------|--------|---------------|-------|
| Skill instructions (SKILL.md files) | This public repo | Public | No sensitive data — these are generic templates |
| Team work items (issues, boards) | GitHub | Internal | Scoped by repo access. No PII. |
| Meeting notes and action items | Generated by agent | Internal | Saved to SharePoint (tenant-protected) |
| Budget/PO data | SharePoint Excel files | Confidential | Access scoped to authorized users only |
| Dashboard metrics | Power BI | Internal | Governed by Power BI RLS (Row-Level Security) |
| Calendar and email | Work IQ MCP | Confidential | Governed by M365 permissions — agents see only what the user can see |

### What the agents CANNOT access

| Data Type | Why Not |
|-----------|---------|
| Other users' email/calendar (without permission) | Work IQ respects M365 permissions |
| HR/personnel records | Not connected as a knowledge source |
| Salary or compensation data | Not connected as a knowledge source |
| Source code or engineering repos | GitHub MCP scoped to marketing repos only |
| External customer PII | Not stored in any connected system |

## Permission Model

### Principle of Least Privilege

Each agent and flow gets only the permissions it needs:

| Agent | What It Accesses | What It Can Write |
|-------|-----------------|-------------------|
| Marketing Assistant | Uploaded SKILL.md files (public), SharePoint docs | Draft documents (with user review) |
| Strategy Advisor | SKILL.md files, web search results | Excel log entries (competitive scan results) |
| Program Tracker | GitHub issues (scoped to marketing repo), SharePoint Excel | Update Excel trackers, post to Teams channels |
| Reporting Engine | GitHub issues, Power BI dashboards, SharePoint | Post to Teams channels, create calendar events, create SharePoint list items |

### Flow Permissions

Each Power Automate flow uses a dedicated connection with minimal scope:

| Flow | Connection | Scope |
|------|-----------|-------|
| Post to Teams Channel | Microsoft Teams | Specific team + channel only |
| Log to Excel | SharePoint | Specific document library only |
| Send Email | Outlook | Send-as authenticated user only |
| Create Calendar Event | Outlook | Authenticated user's calendar only |
| Create SharePoint List Item | SharePoint | Specific list only |
| Update Excel Tracker | SharePoint | Specific file only |

## Data Residency

| Component | Data Location | Governed By |
|-----------|--------------|------------|
| Copilot Studio agent config | M365 tenant region | Microsoft data residency commitments |
| Power Automate flow data | M365 tenant region | Power Platform data policies |
| SharePoint files | M365 tenant region | SharePoint admin controls |
| GitHub data | GitHub cloud (US) | GitHub enterprise terms |
| MCP server connections | Depends on server | See individual MCP docs |

## Admin Controls

### Copilot Studio Admin Center

- Control which users can create agents
- Control which connectors and MCP servers are available
- Set data loss prevention (DLP) policies
- Monitor agent usage and analytics

### Power Platform Admin Center

- Manage Power Automate flow permissions
- Set environment-level security
- Control connector availability
- Audit flow execution logs

### Microsoft Defender

- Work IQ MCP servers are monitored by Microsoft Defender
- Audit trails for all agent actions
- Policy enforcement on MCP tool invocations
- Suspicious activity alerts

## Incident Response

If an agent behaves unexpectedly:

1. **Immediate:** Remove the agent from Teams (Copilot Studio > Channels > Remove from Teams)
2. **Investigate:** Check agent analytics in Copilot Studio for the triggering query
3. **Fix:** Update instructions or knowledge sources
4. **Prevent:** Add guardrails to instructions ("Never do X")
5. **Report:** Contact your IT admin if data exposure is suspected

## Compliance

| Standard | Coverage |
|----------|---------|
| Microsoft 365 compliance | Agents deployed via Copilot Studio inherit M365 compliance posture |
| Data Loss Prevention (DLP) | Power Platform DLP policies apply to all flows and connectors |
| Audit logging | All agent interactions logged in M365 audit log |
| Entra ID conditional access | Agent access governed by the same conditional access policies as other M365 apps |
| GDPR/privacy | No PII processing by default. If PII is added to knowledge sources, standard M365 GDPR controls apply |

## Checklist Before Deploying

- [ ] Agent authentication set to Microsoft Entra ID (not "No authentication")
- [ ] No confidential documents uploaded as knowledge
- [ ] Power Automate flows scoped to minimum required permissions
- [ ] MCP servers authorized by IT admin
- [ ] Agent tested with 10+ real queries
- [ ] DLP policies reviewed in Power Platform admin center
- [ ] Deployment reviewed with IT security (if required by your org)
- [ ] Team notified about the new agent and its capabilities
