# Deck Builder — Slide Templates

Use the template that best matches the deck type. Each template includes slide-by-slide structure with titles, body content, speaker notes guidance, and visual suggestions.

---

## Stakeholder Briefing (5-7 slides)

Best for: Executive updates, program reviews, initiative check-ins. Audience: VP/Director level.

```
Slide 1: TITLE
  Title: [Initiative/Program Name]
  Subtitle: Briefing for [Audience] | [Date]
  Notes: [Context for presenter — why this briefing matters right now]

Slide 2: EXECUTIVE SUMMARY
  Title: At a Glance
  Body:
    - Status: [Green/Yellow/Red]
    - Key metric: [number with context]
    - Key metric: [number with context]
    - Key decision needed: [if any]
  Notes: Lead with the headline — what does leadership need to know in 10 seconds?

Slide 3: PROGRESS
  Title: What We've Accomplished
  Body:
    - [Win 1 with metric]
    - [Win 2 with metric]
    - [Win 3 with metric]
  Visual: Bar chart or timeline showing progress
  Notes: Celebrate wins first — frame positively before introducing risks.

Slide 4: CURRENT STATE
  Title: Where We Are Now
  Body:
    - In Progress: [items]
    - Blocked: [items with owners]
    - Coming Next: [items]
  Visual: Kanban or pipeline view
  Notes: Be honest about blockers — leadership respects transparency over optimism.

Slide 5: RISKS & ASKS
  Title: What We Need
  Body:
    - Risk 1: [description] -> Ask: [what you need]
    - Risk 2: [description] -> Ask: [what you need]
  Notes: Every risk should have a proposed mitigation and a clear ask. Never present a problem without a recommendation.

Slide 6: NEXT STEPS
  Title: Looking Ahead
  Body:
    - [Next milestone with date]
    - [Next milestone with date]
    - [Decision point with date]
  Notes: End with forward momentum, not problems. Leave leadership feeling confident.

Slide 7: APPENDIX (optional)
  Title: Supporting Data
  Body: [Detailed metrics, comparison tables, backup slides]
  Notes: This is where you put the data your VP might ask about. Always have backup data ready — it shows preparation even if nobody asks.
```

## Monthly Business Review (8-10 slides)

Best for: Recurring monthly reporting cadence. Audience: Sr. Director+ level.

```
Slide 1: TITLE
  Title: Monthly Business Review — [Month Year]
  Subtitle: [Team/Function Name]
  Notes: MBR should tell a story: what happened, what it means, what's next.

Slide 2: EXECUTIVE SUMMARY
  Title: This Month in 3 Headlines
  Body:
    - [Headline 1 — biggest win or shift]
    - [Headline 2 — key metric movement]
    - [Headline 3 — what's coming / risk flag]
  Notes: If leadership reads only this slide, they should get the full picture.

Slide 3: WINS
  Title: What We Shipped
  Body:
    - [Completed item with impact metric]
    - [Completed item with impact metric]
    - [Completed item with impact metric]
  Visual: Checkmark list or completion tracker
  Notes: "Shipped" means delivered to audience, not "worked on." Be rigorous.

Slide 4: BY THE NUMBERS
  Title: Key Metrics
  Body:
    - [Metric 1]: [value] (vs. [target] / [last month] / [last year])
    - [Metric 2]: [value] (vs. comparison)
    - [Metric 3]: [value] (vs. comparison)
  Visual: Bar or line chart showing month-over-month trends
  Notes: Every metric needs context. A number alone means nothing — compare to target, prior period, or competitor.

Slide 5: BY TEAM MEMBER
  Title: Team Delivery
  Body:
    - [Person 1]: [Key deliverable]
    - [Person 2]: [Key deliverable]
    - [Person 3]: [Key deliverable]
  Notes: Shows team breadth and individual contributions. Skip if audience doesn't need this level of detail.

Slide 6: PROGRAM DEEP-DIVE
  Title: Spotlight: [Program Name]
  Body:
    - Goal: [what we set out to do]
    - Result: [what happened with metrics]
    - Insight: [what we learned]
  Visual: Before/after comparison, funnel, or journey map
  Notes: Pick 1-2 programs that had interesting results this month. Rotate monthly.

Slide 7: PIPELINE
  Title: Coming Next Month
  Body:
    - [Initiative 1] — [milestone and date]
    - [Initiative 2] — [milestone and date]
    - [Initiative 3] — [milestone and date]
  Notes: Forward-looking. What should leadership expect to hear about next month?

Slide 8: RISKS & BLOCKERS
  Title: Needs Leadership Attention
  Body:
    - [Risk 1]: [Impact] -> [Mitigation] -> [Ask]
    - [Risk 2]: [Impact] -> [Mitigation] -> [Ask]
  Notes: Only include risks that are strategic-level. Day-to-day fires belong in your team standup, not here.

Slide 9: ASKS
  Title: Decisions & Approvals Needed
  Body:
    - [Ask 1]: [What, why, deadline]
    - [Ask 2]: [What, why, deadline]
  Notes: Make it easy to say yes. Include all context needed to decide in this slide.

Slide 10: APPENDIX
  Title: Supporting Data
  Body: [Detailed data tables, full metric breakdowns, backup analysis]
```

## Event / Program One-Pager (2-3 slides)

Best for: Quick overview of a specific event or program. Audience: Anyone who needs the "what is this?" summary.

```
Slide 1: OVERVIEW
  Title: [Event/Program Name]
  Body:
    - What: [1-line description]
    - When: [date(s)]
    - Who: [audience and expected attendance]
    - Goal: [target metric]
    - Status: [Green/Yellow/Red]
  Notes: One-pager should answer the 5 W's in a single glance.

Slide 2: DETAILS
  Title: How It Works
  Body:
    - Program structure / agenda
    - Partners involved
    - Budget summary (approved / remaining)
    - Key timeline milestones
  Visual: Timeline or Gantt-style view
  Notes: This slide is for people who want to go one level deeper without reading the full planning doc.

Slide 3: ASK (include only if needed)
  Title: What We Need
  Body:
    - [Specific ask from leadership]
    - [Decision deadline]
    - [Impact of delay]
  Notes: Only include this slide if there's an active ask. Otherwise, 2 slides is perfectly fine.
```

## Custom / Freeform Deck

When none of the above templates fit, build a custom deck using these structural rules:

1. **Slide 1 is always a title slide** with topic, audience, and date
2. **Slide 2 is always the executive summary** — if someone reads only this slide, they get the message
3. **Middle slides follow a logical narrative arc**: situation -> action -> result, or problem -> analysis -> recommendation
4. **Second-to-last slide is the ask** — what you need from the audience
5. **Last slide is an appendix** (optional) — backup data for Q&A
